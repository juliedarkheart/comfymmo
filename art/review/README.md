# Asset review (contact sheets)

Human-review artifacts — **not used by the game**. Generated by
`tools/art/make_asset_contact_sheet.py`:

- `generated_terrain_contactsheet.png` — current cozy terrain tiles + labels.
- `generated_objects_contactsheet.png` — current object/prefab sprites + labels.
- `generated_icons_contactsheet.png` — current UI/tool icons + labels.
- `<source>_<pack>_contactsheet.png` — an imported pack. **Unlabeled** sheets get
  a numbered grid overlay (cell indices) so you can pick cells by number;
  **named** packs keep their names.

## Workflow

1. Look at the contact sheet (and run `tools/art/asset_preview.tscn` in the
   editor for an in-engine view).
2. Decide which specific cells/sprites are worth using.
3. Slice + normalize them into `art/generated/from_external/active/<mirror>`
   (`tools/art/slice_spritesheet.py`), recording them in
   `art/generated/from_external/manifest.json`.
4. Activate by listing them in `art/active_art_manifest.json`.

Nothing here is auto-wired. Unlabeled terrain sheets are never activated
automatically. See `docs/asset_review_workflow.md`.
